Ausführliche Beschreibung

Der TaskScheduler ist für das Ausführen von Tasks verantwortlich.

Ein Task kann an die Methode schedule() übergeben werden und reiht sich dann in eine Warteschlange ein, die Regelmäßig nach den Prioritäten der Tasks sortiert wird.

Werden Resourcen (sprich Threads) frei, startet der TaskScheduler weitere Tasks aus seiner Warteschlange, solange bis keine Threads mehr zur Verfügung stehen, oder keine Tasks mehr vorhanden sind.

Der TaskScheduler ist dabei ein eigener Thread, sodass er nicht warten muss, falls die GUI gerade auf irgendetwas wartet.

Außerdem ist TaskScheduler ein Singleton, sodass von überall aus auf ihn zugegriffen werden kann.
